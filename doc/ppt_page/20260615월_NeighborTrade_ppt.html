01 INTRO

프로젝트 소개

NeighborTrade는 우리 동네 안에서 물건을 사고팔고, 이웃과 정보를 나누는 위치 기반 웹 플랫폼입니다.

핵심 가치

  • 우리 동네 이웃과의 안전·투명한 직거래
  • 비로그인 공개 탐색 → 가입·동네 인증 전환
  • 중고 + 동네생활 + 운영 거버넌스
서비스 대상
  • 동네에서 물건을 사고팔고 싶은 개인
  • 안전한 중고·지역 정보를 원하는 사용자
  • 지역 커뮤니티에 관심 있는 주민

07b OPENAPI

OpenAPI 3.0 적용 · 왜 REST + 문서화?

NeighborTrade는 Thymeleaf SSR이 주력이지만, sidebar·region-modal 등 JSON API가 함께 동작한다. OpenAPI는 이 REST 계층의 단일 진실 공급원(SSOT) 이다.

적용 목적

  • 프론트(region-modal.js) ↔ 서버 계약 명세
  • 발표·포트폴리오 Try it out 데모
  • Phase 4+ REST 확장 시 스펙 선행
페이지 vs API 분리
  • Thymeleaf — /market/*, /community/* (OpenAPI 제외)
  • REST — /api/* (OpenAPI 문서화)
  • WebSocket — /ws STOMP (별도 프로토콜)
springdoc-openapi 2.x Swagger UI JSON + YAML

    .info(new Info().title("NeighborTrade API")

}

// SecurityConfig — swagger permitAll
.requestMatchers("/swagger-ui/**", "/v3/api-docs/**").permitAll()
